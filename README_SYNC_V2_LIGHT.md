# DAWAA Sync v2 Light

هذا الإصدار يصلح التزامن بدون بطء وبدون Polling ثقيل.

## ماذا تغير؟

- عند فتح صفحة الضيوف أو بوابة العميل، يتم تحميل الضيوف من Supabase مرة واحدة.
- عند إضافة أو تعديل ضيف، يتم حفظه في Supabase مباشرة في الخلفية.
- عند استيراد قائمة، يتم حفظ الضيوف في Supabase مباشرة.
- تمت إزالة السحب كل 10 ثواني الذي سبب البطء.
- تمت المحافظة على ميزة البطاقات المتعددة V2.

## الملفات التي يتم استبدالها فقط

- app.js
- api/guests-sync.js
- api/_lib/supabase.js
- api/_lib/meta.js
- api/webhook/meta.js

## طريقة الاختبار

1. ارفعي الملفات واعملي Commit.
2. انتظري Vercel Deploy.
3. من اللابتوب: أضيفي ضيف جديد.
4. انتظري ثانيتين.
5. من الجوال: افتحي صفحة المدعوين واضغطي تحديث البيانات إذا لزم.
6. يجب أن يظهر الضيف.
7. جربي ضيف عنده 3 بطاقات للتأكد أن منطق البطاقات المتعددة ما زال يعمل.

## ملاحظة

لا يوجد تزامن لحظي فوري الآن. هذا مقصود حتى لا يصير الموقع بطيئًا.
التزامن يتم عند فتح الصفحة، أو عند الضغط على زر "تحديث البيانات"، أو بعد الإضافة/التعديل.
